# Build & Deploy — Brew customer app (iOS · Android · Web/PWA)

One Flutter/Dart codebase, three targets. The UI is **mobile-adaptive** (full-bleed
on phones, centered phone-width canvas on web/desktop via `ResponsiveShell`), and
the web build is an **installable PWA**.

## 0. One-time: generate native project scaffolding
This repo holds the Dart source (`lib/`), the PWA web assets (`web/`), tests, and
`pubspec.yaml`. The native iOS/Android (and default web) projects are generated by
Flutter — run this once after cloning (it fills in missing platform folders and
does **not** overwrite existing `lib/`, `web/`, or `pubspec.yaml`):

```bash
cd apps/brew-mobile-customer
flutter create . \
  --org com.projectbrew \
  --project-name brew_mobile_customer \
  --platforms=android,ios,web
flutter pub get
```

This creates `android/`, `ios/`, and any missing `web/` assets (icons, favicon,
`flutter_bootstrap.js` at build time). The app/bundle id becomes
`com.projectbrew.brew_mobile_customer` (change `--org` to your domain).

> The provided `web/index.html` + `web/manifest.json` are kept (Brew theming +
> PWA). `flutter create` won't clobber them.

## 1. Configure the API endpoint
All targets read the backend base + store from `--dart-define`:
```bash
--dart-define=BREW_API=https://api.brew.example   # backend base URL
--dart-define=BREW_STORE=store_1                  # store this client browses
```
Local dev: Android emulator uses `http://10.0.2.2:3000`; iOS sim / web use
`http://localhost:3000`.

## 2. Web / PWA
```bash
# run locally
flutter run -d chrome --dart-define=BREW_API=http://localhost:3000

# production build → build/web (static, installable PWA with service worker)
flutter build web --release --dart-define=BREW_API=https://api.brew.example
```
Deploy `build/web/` to any static host. In this platform that's the
`brew-infra` **cdn** module (S3 + CloudFront) — e.g.:
```bash
aws s3 sync build/web s3://<pwa-bucket> --delete
aws cloudfront create-invalidation --distribution-id <id> --paths '/*'
```
The PWA is installable (Add to Home Screen) on Android/desktop and iOS Safari.

### Live preview on GitHub Pages (automatic)
`.github/workflows/deploy-pwa-pages.yml` publishes the web/PWA build to GitHub
Pages on every push to `main`. **One-time setup:** repo **Settings → Pages →
Source = "GitHub Actions"**. Live URL: `https://<owner>.github.io/<repo>/`
(here: `https://abhilashyes.github.io/cafe-system/`).

Set a repo **Variable** `BREW_API` (Settings → Secrets and variables → Actions →
Variables) to your public backend URL so the preview's API calls resolve; the
build injects it and sets `--base-href /<repo>/` automatically.

## 3. Android → Google Play
```bash
# 1) set applicationId (defaults to com.projectbrew.brew_mobile_customer) in
#    android/app/build.gradle
# 2) create an upload keystore and android/key.properties (NEVER commit it):
#    storeFile/storePassword/keyAlias/keyPassword
# 3) build the App Bundle
flutter build appbundle --release --dart-define=BREW_API=https://api.brew.example
# → build/app/outputs/bundle/release/app-release.aab
```
Upload the `.aab` in the Play Console (internal testing → production). Required:
app icon, privacy policy URL (DPDP), data-safety form, screenshots.

## 4. iOS → App Store
```bash
# 1) open ios/Runner.xcworkspace, set Bundle Identifier + Signing Team
# 2) build a signed archive
flutter build ipa --release --dart-define=BREW_API=https://api.brew.example
# → build/ios/ipa/*.ipa
```
Upload via Xcode Organizer or **Transporter** to App Store Connect. Required:
bundle id registered, App Store icon, privacy nutrition labels, screenshots.

## 5. App icons (optional, recommended)
Use `flutter_launcher_icons` once you have a 1024×1024 PNG at `assets/icon.png`:
```yaml
# add to dev_dependencies, then run: dart run flutter_launcher_icons
flutter_launcher_icons:
  image_path: "assets/icon.png"
  android: true
  ios: true
  web: { generate: true }
```

## 6. Versioning
Bump `version: x.y.z+build` in `pubspec.yaml` for every store submission
(`x.y.z` = user-facing, `+build` = monotonically increasing build number).

## Notes
- Secure token storage (Keychain/Keystore), SSL pinning, and root/jailbreak
  awareness should be added before store release (platform threat model).
- CI: `.github/workflows/flutter.yml` runs `flutter analyze` + `flutter test` and
  builds the web bundle on every change to this app.
